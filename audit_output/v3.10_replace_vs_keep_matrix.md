# TaxFlow Pro v3.10 — Replace vs. Keep Decision Matrix

## Purpose
Identify ground-up TaxFlow Pro components that could be replaced by mature, prebuilt infrastructure in the v3.10 build/packaging release.

## Scoring legend
| Factor | Weight | Meaning |
|--------|--------|---------|
| Offline risk | High | Does the replacement break offline-first requirement? |
| Maintenance burden | High | Will the replacement reduce long-term maintenance? |
| Security / correctness | High | Is the replacement more secure or correct than custom code? |
| Build size impact | Medium | Does the replacement balloon installer size? |
| Migration effort | Medium | How much existing code/tests/docs must change? |
| Strategic fit | High | Does it align with v3.10 packaging goal? |

## Verdict summary

| Verdict | Components |
|---------|------------|
| **Replace now (v3.10)** | Desktop packaging (PyInstaller), backup encryption (`age`), static frontend serving |
| **Simplify / keep** | Secret storage, RLS/tenant isolation, rate limiting, audit trail |
| **Defer to later spike** | PDF/OCR replacement (`marker-pdf`/`docling`), ML categorizer ONNX migration |
| **Add new** | Auto-launch browser, first-run wizard frontend, optional tray icon |

---

## Recommended v3.10 scope

1. PyInstaller build pipeline.
2. FastAPI static mount of Vite-built frontend.
3. User-writable data directory abstraction.
4. `age`-based backup/encrypt + restore.
5. Auto-launch browser on first run.
6. Optional: simplify RLS layer for single-user SQLite builds.
7. Optional: first-run wizard UI.
