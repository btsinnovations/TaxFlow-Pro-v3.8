# TaxFlow Pro v3.10 Orchestration Plan

## Goal

Transform TaxFlow Pro from a v3.9.2 local-first tax pipeline into a full small-business bookkeeping platform, while remaining offline-first and packaged later (v3.10.5).

## Deadline & cadence

- **Target:** 30 days from start date.
- **Check-ins:** Daily with Josh.
- **Release model:** One big v3.10 release, no staged alpha/beta unless scope slips.

## Core principles

1. **Offline-first.** No live bank feeds, no cloud auth, no real-time FX APIs.
2. **Clean-slate data model.** New v3.10 profiles start fresh; v3.9.2 backup import is deferred.
3. **All new modules must have tests.**
4. **Unified register-style UI.** Frontend centerpiece for transaction management.

---

## Frontend architecture

### UI stack
- **Component library:** shadcn/ui + Tailwind CSS (already present via Vite).
- **Data grid:** TanStack Table v8 for inline register editing, sorting, filtering, bulk selection.
- **Charts/reports:** Recharts for cash flow / budget visualizations; html2canvas + jsPDF for PDF export.

### New views
1. **Register** — unified transaction register per account.
2. **Accounts** — COA tree with sub-accounts, balances, drill-down.
3. **Account detail** — tabs for transactions, loans, investments, inventory.
4. **Recurring rules** — list + create/edit recurrence.
5. **Reconciliation** — import, match, clear transactions.
6. **Tax exports** — per-tax-year export to TurboTax, H&R Block, Drake formats.
7. **Reports center** — P&L, balance sheet, cash flow, in-app preview + PDF export.
8. **Check register** — printable and on-screen register.
9. **Inventory center** — items, valuation method, quantity, COGS transactions.
10. **Multi-currency** — rate table, foreign-amount entry.
11. **Role management** — profile membership admin.

## Data model changes

### New tables
- `profile_memberships` — user_id, profile_id, role.
- `coa_accounts` — id, number, name, type, parent_id, is_active.
- `recurring_rules` — account_id, amount, description, frequency, start_date, end_date, count, splits JSON.
- `inventory_items` — id, sku, name, cogs_account_id, income_account_id, valuation_method, qty_on_hand.
- `inventory_transactions` — item_id, qty, unit_cost, valuation_method snapshot.
- `fx_rates` — from_currency, to_currency, rate, effective_date.
- `loan_schedules` — account_id, schedule_type, original_principal, rate, term, start_date.
- `investment_lots` — account_id, shares, cost_basis, acquisition_date.
- `reconciliation_imports` — id, account_id, import_date, statement_balance.
- `reconciliation_matches` — import_id, transaction_id, match_type.
- `budget_lines` — category_id, period, amount.

### Migrations
- Single Alembic migration to replace `accounts`/`gl_accounts` with `coa_accounts` while preserving existing data.
- Add new tables.
- Update `transactions` table to support splits JSON, foreign amount, project tags.
