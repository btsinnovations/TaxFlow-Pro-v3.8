# TaxFlow Pro v3.11 — Bookkeeping Platform Expansion Spec

**Status:** Updated per 2026-06-25 directive  
**Branch:** `v3.11-dev`  
**Goal:** Transform TaxFlow Pro from a local-first tax-export assistant into a full small-business bookkeeping platform, while integrating all existing bank parsers and deferred v3.10 UI/UX items.

---

## 1. Core Principles

1. **Offline-first.** No live bank feeds that require cloud auth by default; OFX file import is the primary integration path.
2. **Clean-slate data model.** New v3.11 profiles start fresh; v3.10 backup import is the migration path.
3. **All new modules must have tests.**
4. **Unified register-style UI.** Frontend centerpiece for transaction management.
5. **Feature-forward release.** Security hardening and desktop packaging are deferred to v3.11.5.

## 3. Newly Added v3.11 Scope

### 3.1 Bank Parser Expansion (3.11.PARSER)

Integrate **all currently available bank parsers** into the v3.11 release.

- Inventory existing parsers under `backend/parsers/` and any parser specs.
- Normalize parser interface: `parse(file_bytes, filename, hint=None) -> ParsedStatement`.
- Add parser auto-detection endpoint and UI hint.
- Ensure every parser has at least one test fixture and passing tests.
- Update Hero/branding count to reflect actual institution coverage.

### 3.2 Tax Rules Engine Search/Filter (3.11.TAXRULES)

Add search and filter capabilities to the categorization/tax rules engine.

- API: `GET /api/tax-rules?search={text}&category={id}&account={id}&priority_min={n}&priority_max={n}`.
- UI: search bar + category/account/priority filters in the rules management screen.
- Sort by priority, created date, pattern length.
- Tests for backend filtering and frontend integration.

### 3.3 OFX / Live Feed Integration (3.11.OFX)

- OFX file import route: `POST /api/imports/ofx`.
- Parse OFX statements into the same internal transaction model used by PDF parsers.
- Optional "live feed" path: local OFX server polling (no cloud auth; user-configured endpoint).
- Map OFX accounts to TaxFlow accounts; reconcile duplicates by `FITID`.
- Tests for OFX parsing, duplicate handling, and endpoint validation.

### 3.4 Polished Export UI (3.11.EXPORTUI)

Address the v3.10 handoff items now.

- Output-formats dropdown: remove stale "(HomeBank)" label from QIF.
- Enable format selection only when processed statements exist; show disabled hint otherwise.
- Add clear export status/progress indicator.
- Tests for export availability state and dropdown contents.

## 4. Data Model Changes

### New Tables

- `profile_memberships` — user_id, profile_id, role.
- `coa_accounts` — id, number, name, type, parent_id, is_active.
- `recurring_rules` — account_id, amount, description, frequency, start_date, end_date, count, splits JSON.
- `inventory_items` — id, sku, name, cogs_account_id, income_account_id, valuation_method, qty_on_hand.
- `inventory_transactions` — item_id, qty, unit_cost, valuation_method snapshot.
- `fx_rates` — from_currency, to_currency, rate, effective_date.
- `loan_schedules` — account_id, schedule_type, original_principal, rate, term, start_date.
- `investment_lots` — account_id, shares, cost_basis, acquisition_date.
- `reconciliation_imports` — id, account_id, import_date, statement_balance.
- `reconciliation_matches` — import_id, transaction_id, match_type.
- `budget_lines` — category_id, period, amount.
- `tax_rules` — updated to support search indexing columns if needed.
- `ofx_imports` — id, account_id, import_date, as_of_date, file_hash.

### Migrations

- Single Alembic migration to replace `accounts`/`gl_accounts` with `coa_accounts` while preserving existing data.
- Add new tables.
- Update `transactions` table to support splits JSON, foreign amount, project tags.
- Add `fitid` column to transactions for OFX duplicate detection.

## 5. Frontend Architecture

- **Component library:** shadcn/ui + Tailwind CSS.
- **Data grid:** TanStack Table v8 for inline register editing, sorting, filtering, bulk selection.
- **Charts/reports:** Recharts for cash flow / budget visualizations; html2canvas + jsPDF for PDF export.
- **Navigation order:** Dashboard → Upload → Processed Files → Register → Reports.
