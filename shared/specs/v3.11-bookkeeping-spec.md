# TaxFlow Pro v3.11 ŌĆö Bookkeeping Platform Expansion Spec

**Status:** Spec complete; implementation begins after v3.10 tooling/security hardening ships  
**Branch:** `v3.11-dev` (cut from v3.10 release tag)  
**Goal:** Transform TaxFlow Pro from a local-first tax-export assistant into a full small-business bookkeeping platform.

---

## 1. Core Principles

1. **Offline-first.** No live bank feeds, no cloud auth, no real-time FX APIs.
2. **Clean-slate data model.** New v3.11 profiles start fresh; v3.10 backup import is the migration path.
3. **All new modules must have tests.**
4. **Unified register-style UI.** Frontend centerpiece for transaction management.

## 3. Data Model Changes

### New Tables

- `profile_memberships` ŌĆö user_id, profile_id, role.
- `coa_accounts` ŌĆö id, number, name, type, parent_id, is_active.
- `recurring_rules` ŌĆö account_id, amount, description, frequency, start_date, end_date, count, splits JSON.
- `inventory_items` ŌĆö id, sku, name, cogs_account_id, income_account_id, valuation_method, qty_on_hand.
- `inventory_transactions` ŌĆö item_id, qty, unit_cost, valuation_method snapshot.
- `fx_rates` ŌĆö from_currency, to_currency, rate, effective_date.
- `loan_schedules` ŌĆö account_id, schedule_type, original_principal, rate, term, start_date.
- `investment_lots` ŌĆö account_id, shares, cost_basis, acquisition_date.
- `reconciliation_imports` ŌĆö id, account_id, import_date, statement_balance.
- `reconciliation_matches` ŌĆö import_id, transaction_id, match_type.
- `budget_lines` ŌĆö category_id, period, amount.

### Migrations

- Single Alembic migration to replace `accounts`/`gl_accounts` with `coa_accounts` while preserving existing data.
- Add new tables.
- Update `transactions` table to support splits JSON, foreign amount, project tags.

## 4. Frontend Architecture

- **Component library:** shadcn/ui + Tailwind CSS.
- **Data grid:** TanStack Table v8 for inline register editing, sorting, filtering, bulk selection.
- **Charts/reports:** Recharts for cash flow / budget visualizations; html2canvas + jsPDF for PDF export.
